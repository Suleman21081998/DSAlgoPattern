Segment Tree Declaration and Initialization for Competitive Programming

Definition
- A segment tree is a tree data structure used to store information about array intervals.
- Each node stores a value for a segment of the array.
- It is commonly used for range queries and range updates.
- Typical operations include sum, minimum, maximum, and gcd over a range.

Important Idea: Internal Working
- The tree is built over an array using recursion or iterative techniques.
- Each node represents a segment, and its value is computed from its children.
- A query over a range visits only O(log n) nodes.
- Updates also take O(log n) time because only the nodes along the update path change.

- Time complexity:

  - Point update: O(log n)
  - Range query: O(log n)

- Proper class example in C++:
  class SegmentTree {
  private:
      vector<int> tree;
      int n;

      void build(const vector<int>& arr, int node, int left, int right) {
          if (left == right) {
              tree[node] = arr[left];
              return;
          }
          int mid = (left + right) / 2;
          build(arr, node * 2, left, mid);
          build(arr, node * 2 + 1, mid + 1, right);
          tree[node] = tree[node * 2] + tree[node * 2 + 1];
      }

      void update(int node, int left, int right, int pos, int value) {
          if (left == right) {
              tree[node] = value;
              return;
          }
          int mid = (left + right) / 2;
          if (pos <= mid) update(node * 2, left, mid, pos, value);
          else update(node * 2 + 1, mid + 1, right, pos, value);
          tree[node] = tree[node * 2] + tree[node * 2 + 1];
      }

      int query(int node, int left, int right, int ql, int qr) {
          if (ql <= left && right <= qr) return tree[node];
          if (qr < left || right < ql) return 0;
          int mid = (left + right) / 2;
          return query(node * 2, left, mid, ql, qr) + query(node * 2 + 1, mid + 1, right, ql, qr);
      }

  public:
      SegmentTree(const vector<int>& arr) {
          n = arr.size();
          tree.assign(4 * n + 5, 0);
          build(arr, 1, 0, n - 1);
      }

      void update(int pos, int value) {
          update(1, 0, n - 1, pos, value);
      }

      int query(int l, int r) {
          return query(1, 0, n - 1, l, r);
      }
  };

- Proper class example in Python:
  class SegmentTree:
      def __init__(self, arr):
          self.n = len(arr)
          self.tree = [0] * (4 * self.n + 5)
          self.build(arr, 1, 0, self.n - 1)

      def build(self, arr, node, left, right):
          if left == right:
              self.tree[node] = arr[left]
              return
          mid = (left + right) // 2
          self.build(arr, node * 2, left, mid)
          self.build(arr, node * 2 + 1, mid + 1, right)
          self.tree[node] = self.tree[node * 2] + self.tree[node * 2 + 1]

      def update(self, pos, value):
          self._update(1, 0, self.n - 1, pos, value)

      def _update(self, node, left, right, pos, value):
          if left == right:
              self.tree[node] = value
              return
          mid = (left + right) // 2
          if pos <= mid:
              self._update(node * 2, left, mid, pos, value)
          else:
              self._update(node * 2 + 1, mid + 1, right, pos, value)
          self.tree[node] = self.tree[node * 2] + self.tree[node * 2 + 1]

      def query(self, l, r):
          return self._query(1, 0, self.n - 1, l, r)

      def _query(self, node, left, right, ql, qr):
          if ql <= left and right <= qr:
              return self.tree[node]
          if qr < left or right < ql:
              return 0
          mid = (left + right) // 2
          return self._query(node * 2, left, mid, ql, qr) + self._query(node * 2 + 1, mid + 1, right, ql, qr)

Use Cases
- Range sum queries
- Range minimum/maximum queries
- Range updates with point updates
- Competitive programming problems involving prefix/suffix operations
- Dynamic array queries where many updates and queries happen

Competitive Programming Tips
- Segment tree is ideal when you need fast range queries after many updates.
- It is more flexible than prefix sums because it supports updates efficiently.
- For static arrays with no updates, prefix sums may be simpler.
